Plugin Header

Every plugin must include a header comment in its main file. This provides metadata that VoidForge uses to display plugin information in the admin area. 

<?php
/**
 * Plugin Name: My Awesome Plugin
 * Description: A short description of what your plugin does.
 * Version: 1.0.0
 * Author: Your Name
 * Author URI: https://yoursite.com
 * Requires PHP: 8.0
 * Requires CMS: 0.1.2
 */

defined('CMS_ROOT') or die;

// Your plugin code here...
Header Field Required Description
Plugin Name Yes The display name of your plugin
Description Yes Brief description (shown in admin)
Version Yes Semantic version number (e.g., 1.0.0)
Author No Plugin author name
Author URI No Author website URL
Requires PHP No Minimum PHP version required
Requires CMS No Minimum VoidForge version required

Security First

Always include defined('CMS_ROOT') or die; at the top of your plugin files to prevent direct access.

Your First Plugin

Let's create a simple "Hello World" plugin that adds a shortcode and an admin page.

Step 1: Create the Plugin Folder

Create a new folder called hello-world inside the /plugins directory.

Step 2: Create the Main Plugin File

Create hello-world.php inside your new folder:

<?php
/**
 * Plugin Name: Hello World
 * Description: A simple example plugin.
 * Version: 1.0.0
 * Author: Your Name
 */

defined('CMS_ROOT') or die;

// Add a shortcode: [hello name="World"]
add_shortcode('hello', function($atts) {
    $name = $atts['name'] ?? 'World';
    return '<p class="hello-greeting">Hello, ' . esc($name) . '!</p>';
});

// Add an admin page
add_admin_page('hello-world', [
    'title' => 'Hello World',
    'menu_title' => 'Hello World',
    'icon' => 'smile',
    'callback' => function() {
        echo '<h1>Hello World Plugin</h1>';
        echo '<p>Use [hello name="Your Name"] in your posts.</p>';
    }
]);

Step 3: Activate the Plugin

Go to Plugins in the admin dashboard and click Activate on your new plugin.

Success!

Your plugin is now active. Use [hello name="VoidForge"] in any post or page to see it in action.

Hooks & Actions

Hooks allow you to execute code at specific points in VoidForge's execution. Actions are used when you want to do something at a certain point.

Adding an Action

// Execute code when plugins are loaded
add_action('plugins_loaded', function() {
    // Your code runs after all plugins load
    error_log('My plugin is ready!');
});

// With priority (lower = earlier execution)
add_action('init', 'my_init_function', 5);

// With accepted arguments
add_action('post_saved', function($postId, $postData) {
    // Access post data when saved
}, 10, 2);

Triggering Actions

// Trigger an action (for plugin developers)
do_action('my_custom_hook');

// With arguments
do_action('my_custom_hook', $arg1, $arg2);

Removing Actions

// Remove a previously added action
remove_action('init', 'my_init_function', 5);

// Check if action exists
if (has_action('init')) {
    // Action is registered
}

Filters

Filters allow you to modify data as it passes through the system. Unlike actions, filters always return a value. 

// Modify post content before display
add_filter('the_content', function($content) {
    // Add copyright notice to all posts
    return $content . '<p class="copyright">¬© 2024 My Site</p>';
});

// Modify page title
add_filter('document_title', function($title) {
    return $title . ' | My Site';
});

// Apply a filter
$content = apply_filters('the_content', $rawContent);

Shortcodes

Shortcodes let users embed dynamic content using simple [shortcode] syntax in posts and pages.

Self-Closing Shortcode

// Usage: [current_year]
add_shortcode('current_year', function() {
    return date('Y');
});

Shortcode with Attributes

// Usage: [button text="Click Me" url="/page" color="blue"]
add_shortcode('button', function($atts) {
    // Set defaults
    $atts = array_merge([
        'text' => 'Click Here',
        'url' => '#',
        'color' => 'primary'
    ], $atts ?? []);
    
    return sprintf(
        '<a href="%s" class="btn btn-%s">%s</a>',
        esc($atts['url']),
        esc($atts['color']),
        esc($atts['text'])
    );
});

Enclosing Shortcode (with content)

// Usage: [alert type="warning"]Watch out![/alert]
add_shortcode('alert', function($atts, $content = '') {
    $type = $atts['type'] ?? 'info';
    
    return sprintf(
        '<div class="alert alert-%s">%s</div>',
        esc($type),
        do_shortcode($content) // Process nested shortcodes
    );
});

Nested Shortcodes

Use do_shortcode($content) inside enclosing shortcodes to allow nested shortcode processing.

Admin Pages

Add custom pages to the VoidForge admin dashboard for plugin settings and interfaces. 

// Add a page under Plugins menu (default)
add_admin_page('my-settings', [
    'title' => 'My Plugin Settings',
    'menu_title' => 'My Plugin',
    'icon' => 'settings',
    'callback' => 'render_settings_page'
]);

// Add a top-level menu item
add_admin_page('my-dashboard', [
    'title' => 'My Dashboard',
    'parent' => null, // null = top-level
    'icon' => 'layout',
    'callback' => function() {
        // Your page HTML here
        ?>
        <div class="my-plugin-page">
            <h1>My Dashboard</h1>
            <p>Welcome to my plugin!</p>
        </div>
        <?php
    }
]);
Option Type Description
title string Page title (shown in browser tab)
menu_title string Sidebar menu text (defaults to title)
parent string|null 'plugins' (default), null for top-level
icon string Lucide icon name (e.g., 'settings', 'users')
callback callable Function that renders the page content

Settings API

Store and retrieve plugin settings with a simple, type-safe API.

Registering Settings

// Register settings schema
register_plugin_settings('my-plugin', [
    'api_key' => [
        'type' => 'string',
        'default' => '',
        'label' => 'API Key'
    ],
    'enabled' => [
        'type' => 'boolean',
        'default' => true,
        'label' => 'Enable Feature'
    ],
    'max_items' => [
        'type' => 'integer',
        'default' => 10,
        'label' => 'Maximum Items'
    ]
]);

Getting & Setting Values

// Get a setting
$apiKey = get_plugin_setting('my-plugin', 'api_key');
$enabled = get_plugin_setting('my-plugin', 'enabled', true);

// Set a setting
set_plugin_setting('my-plugin', 'api_key', 'sk-12345');
set_plugin_setting('my-plugin', 'enabled', false);

AJAX Handlers

Handle asynchronous requests from your plugin's JavaScript.

PHP Handler

// Register authenticated AJAX handler
register_ajax_handler('my_save_data', function() {
    // Get POST data
    $data = $_POST['data'] ?? null;
    
    if (!$data) {
        wp_send_json_error(['message' => 'No data provided']);
    }
    
    // Process and save...
    
    wp_send_json_success(['message' => 'Saved!']);
});

// Public handler (no login required)
register_ajax_handler('my_public_action', function() {
    wp_send_json_success(['time' => time()]);
}, true); // true = nopriv (public)

JavaScript Request

// Make AJAX request
fetch('/admin/plugin-ajax.php', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/x-www-form-urlencoded'
    },
    body: new URLSearchParams({
        action: 'my_save_data',
        data: JSON.stringify({ key: 'value' }),
        csrf_token: document.querySelector('[name="csrf_token"]').value
    })
})
.then(r => r.json())
.then(data => {
    if (data.success) {
        alert(data.data.message);
    }
});

REST API

Create custom REST API endpoints for headless integrations or complex AJAX operations. 

// Register a REST route
// Accessible at: /api/myplugin/items
register_rest_route('myplugin', 'items', [
    'methods' => ['GET', 'POST'],
    'callback' => function($params) {
        if ($_SERVER['REQUEST_METHOD'] === 'GET') {
            return ['items' => ['a', 'b', 'c']];
        }
        // POST - create item
        return ['created' => true];
    },
    'permission_callback' => function() {
        return User::isLoggedIn();
    }
]);

// Route with URL parameters
// Accessible at: /api/myplugin/items/123
register_rest_route('myplugin', 'items/{id}', [
    'methods' => 'GET',
    'callback' => function($params) {
        $id = $params['id'];
        return ['item' => 'Item ' . $id];
    }
]);

Scheduled Tasks (Cron)

Run background tasks on regular intervals. VoidForge's pseudo-cron runs on page loads. 

// Schedule hourly task
schedule_event('my_hourly_task', 'hourly', function() {
    // Clean up old data, send notifications, etc.
    error_log('Hourly task ran at ' . date('Y-m-d H:i:s'));
});

// Schedule daily task
schedule_event('my_daily_cleanup', 'daily', function() {
    // Daily maintenance...
});

// Available intervals: 'hourly', 'twicedaily', 'daily', 'weekly'

// Unschedule a task
unschedule_event('my_hourly_task');

Widgets

Register widgets that can be displayed in widget areas defined by themes. 

// Register a widget
register_widget('my_recent_posts', [
    'title' => 'Recent Posts',
    'description' => 'Displays recent posts',
    'callback' => function($args) {
        $posts = Post::query([
            'type' => 'post',
            'limit' => $args['count'] ?? 5
        ]);
        
        echo '<ul class="recent-posts">';
        foreach ($posts as $post) {
            echo '<li>' . esc($post['title']) . '</li>';
        }
        echo '</ul>';
    }
]);

Assets (CSS & JavaScript)

Enqueue stylesheets and scripts the proper way.

// Enqueue stylesheet
enqueue_style(
    'my-plugin-style',                    // Handle
    PLUGINS_URL . '/my-plugin/css/style.css', // URL
    [],                                     // Dependencies
    '1.0.0'                                 // Version
);

// Enqueue script
enqueue_script(
    'my-plugin-script',                    // Handle
    PLUGINS_URL . '/my-plugin/js/app.js',   // URL
    [],                                     // Dependencies
    '1.0.0',                                // Version
    true                                    // In footer
);

// Enqueue on init hook
add_action('init', function() {
    enqueue_style('my-style', '/plugins/my-plugin/style.css');
});

Database Helpers

Create and manage custom database tables for your plugin.

// Create a custom table on activation
add_action('activate_my-plugin', function() {
    create_plugin_table('my_custom_data', "
        id INT AUTO_INCREMENT PRIMARY KEY,
        title VARCHAR(255) NOT NULL,
        content TEXT,
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    ");
});

// Drop table on uninstall
add_action('uninstall_my-plugin', function() {
    drop_plugin_table('my_custom_data');
});

// Using the Database class directly
$table = Database::table('my_custom_data');
$items = Database::query("SELECT * FROM {$table} WHERE id = ?", [1]);

Best Practices

Security

  • Always use esc() when outputting user data
  • Validate and sanitize all input data
  • Use verifyCsrf() for form submissions
  • Check user permissions before sensitive operations
  • Use prepared statements with Database::query()

Performance

  • Only load assets on pages that need them
  • Use caching for expensive operations
  • Minimize database queries
  • Use scheduled tasks for heavy processing

Code Quality

  • Use PHP 8+ features (typed properties, named arguments)
  • Follow PSR-12 coding standards
  • Document your code with PHPDoc comments
  • Prefix functions/classes to avoid conflicts
  • Clean up on plugin deactivation/uninstall
