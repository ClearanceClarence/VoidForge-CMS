Theme Development

Theme Development Guide

Learn how to create beautiful, customizable themes for VoidForge CMS with templates, settings, menus, and more.

Introduction

VoidForge themes are collections of PHP templates, CSS styles, and JavaScript files that control how your website looks and behaves. Themes are easy to create and customize without modifying core CMS files.

What You'll Learn

  • Creating theme directory structure
  • Building template files for different content types
  • Adding customizable theme settings
  • Integrating navigation menus
  • Working with custom fields and post types
  • Best practices for theme development
Prerequisites

Basic knowledge of PHP, HTML, and CSS is required. Familiarity with template-based systems is helpful but not required.

Theme Structure

Every theme lives in its own directory inside /themes/. Here's the recommended structure: 

themes/
└── mytheme/
    ├── theme.json          # Theme metadata (required)
    ├── index.php           # Main template (required)
    ├── functions.php       # Theme functions
    ├── header.php          # Header template
    ├── footer.php          # Footer template
    ├── single.php          # Single post template
    ├── page.php            # Page template
    ├── home.php            # Homepage template
    ├── archive.php         # Archive/listing template
    ├── search.php          # Search results template
    ├── 404.php             # Not found template
    ├── welcome.php         # Landing page (optional)
    └── assets/
        ├── css/
        │   └── theme.css
        ├── js/
        │   └── theme.js
        └── img/
            └── logo.svg

Required Files

At minimum, a theme needs:

  • theme.json — Theme metadata and configuration
  • index.php — Fallback template for all content

theme.json

The theme.json file contains metadata about your theme: 

{
    "name": "My Theme",
    "description": "A beautiful custom theme",
    "version": "1.0.0",
    "author": "Your Name",
    "author_url": "https://yoursite.com",
    "screenshot": "screenshot.png",
    "requires_cms": "0.1.4",
    "requires_php": "8.0"
}
Property Required Description
name Yes Display name of the theme
description No Brief description of the theme
version Yes Theme version number
author No Theme author name
screenshot No Preview image filename

Template Hierarchy

VoidForge uses a template hierarchy to determine which file to load. It looks for the most specific template first, then falls back to more general ones:

Single Posts

single-{post_type}.php → single.php → index.php

For a "product" post type: single-product.php → single.php → index.php

Pages

page-{slug}.php → page.php → single.php → index.php

For an "about" page: page-about.php → page.php → index.php

Archives

archive-{post_type}.php → archive.php → index.php

Homepage

home.php → index.php

Search Results

search.php → archive.php → index.php

404 Error

404.php → index.php

Header & Footer

header.php

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title><?= esc($pageTitle ?? getOption('site_name')) ?></title>
    <link rel="stylesheet" href="<?= themeUrl('assets/css/theme.css') ?>">
    <?php do_action('head'); ?>
</head>
<body>
    <header class="site-header">
        <nav>
            <?php echo Menu::display('primary'); ?>
        </nav>
    </header>

footer.php

    <footer class="site-footer">
        <p>&copy; <?= date('Y') ?> <?= esc(getOption('site_name')) ?></p>
        <?php echo Menu::display('footer'); ?>
    </footer>
    <?php do_action('footer'); ?>
</body>
</html>

Including Header & Footer

<?php
get_header();  // Includes header.php
?>

<main>
    <!-- Your content here -->
</main>

<?php
get_footer();  // Includes footer.php
?>

Template Tags

VoidForge provides helper functions for common template tasks:

Site Information

getOption('site_name')      // Site name
getOption('site_tagline')   // Site tagline
siteUrl()                   // Full site URL
themeUrl('path/to/file')   // Theme asset URL

Post Data

$post['title']      // Post title
$post['content']    // Post content (HTML)
$post['excerpt']    // Post excerpt
$post['slug']       // URL slug
$post['created_at'] // Creation date
$post['post_type']  // Post type slug

// Get post URL
getPostUrl($post)

// Get featured image
get_featured_image($post['id'])

Querying Posts

// Get recent posts
$posts = Post::query([
    'post_type' => 'post',
    'status' => 'published',
    'limit' => 10,
    'orderby' => 'created_at',
    'order' => 'DESC'
]);

// Loop through posts
foreach ($posts as $post):
?>
    <article>
        <h2><?= esc($post['title']) ?></h2>
        <div><?= $post['content'] ?></div>
    </article>
<?php endforeach;

Escaping Output

esc($text)        // Escape for HTML output
esc_attr($text)   // Escape for HTML attributes
esc_url($url)     // Escape URL

Theme Settings

Theme settings allow users to customize your theme through the admin panel without editing code.

Getting Settings

<?php
// Get all settings for current theme
$settings = getThemeSettings('mytheme');

// Use settings with defaults
$primaryColor = $settings['primary_color'] ?? '#6366f1';
$showHero = $settings['show_hero'] ?? true;
$heroTitle = $settings['hero_title'] ?? 'Welcome';
?>

<?php if ($showHero): ?>
<section class="hero" style="background: <?= esc($primaryColor) ?>">
    <h1><?= esc($heroTitle) ?></h1>
</section>
<?php endif; ?>

Available Settings

The admin Theme Settings page supports these customization options:

  • Colors — Primary color, secondary color, accent color
  • Hero Section — Toggle, title, subtitle, buttons
  • Features — Up to 6 feature cards with icons
  • Stats — Statistical highlights
  • CTA — Call-to-action section
  • Custom CSS — Additional CSS without file editing

Navigation Menus

VoidForge includes a powerful menu system. Themes can register menu locations and display menus with full customization.

Registering Menu Locations

In your theme's functions.php:

<?php
// Register menu locations
Menu::registerLocation('primary', 'Primary Navigation');
Menu::registerLocation('footer', 'Footer Navigation');
Menu::registerLocation('mobile', 'Mobile Menu');

Displaying Menus

// Simple usage
echo Menu::display('primary');

// With options
echo Menu::display('primary', [
    'container'       => 'nav',
    'container_class' => 'main-navigation',
    'container_id'    => 'site-nav',
    'menu_class'      => 'nav-menu',
    'menu_id'         => 'primary-menu',
    'item_class'      => 'menu-item',
    'link_class'      => 'menu-link',
    'submenu_class'   => 'dropdown-menu',
    'depth'           => 3,  // 0 = unlimited
]);

Menu Display Options

Option Default Description
container 'div' Wrapper element (nav, div, false)
container_class '' CSS class for container
menu_class 'menu' CSS class for ul element
item_class '' CSS class for each li
link_class '' CSS class for each a tag
submenu_class 'sub-menu' CSS class for nested ul
depth 0 Max nesting depth (0 = unlimited)

Example HTML Output

<nav class="main-navigation">
    <ul class="nav-menu">
        <li class="menu-item">
            <a href="/" class="menu-link">Home</a>
        </li>
        <li class="menu-item has-children">
            <a href="/about" class="menu-link">About</a>
            <ul class="dropdown-menu">
                <li class="menu-item">
                    <a href="/team" class="menu-link">Team</a>
                </li>
            </ul>
        </li>
    </ul>
</nav>

Custom Fields

Access custom field values in your templates:

// Get a single field
$price = get_custom_field('price', $post['id']);

// Get all fields for a post
$fields = get_all_custom_fields($post['id']);

// Display in template
if ($price): ?>
    <p class="price">$<?= esc($price) ?></p>
<?php endif;

Scripts & Styles

Theme Asset URLs

// Get URL to theme file
$cssUrl = themeUrl('assets/css/theme.css');
$jsUrl = themeUrl('assets/js/theme.js');
$imgUrl = themeUrl('assets/img/logo.svg');

Enqueuing with Hooks

// In functions.php
add_action('head', function() {
    echo '<link rel="stylesheet" href="' . themeUrl('assets/css/theme.css') . '">';
});

add_action('footer', function() {
    echo '<script src="' . themeUrl('assets/js/theme.js') . '"></script>';
});

Actions & Filters

Use WordPress-style hooks to modify content and add functionality:

Actions

// Add code at specific points
add_action('head', function() {
    echo '<meta name="theme" content="mytheme">';
});

add_action('footer', function() {
    echo '<!-- Custom footer content -->';
});

// Trigger action in template
do_action('my_custom_hook');

Filters

// Modify content
add_filter('the_content', function($content) {
    return $content . '<p>Thanks for reading!</p>';
});

// Modify title
add_filter('the_title', function($title) {
    return strtoupper($title);
});

// Apply filter in template
$content = apply_filters('the_content', $post['content']);

Custom Post Types

Create templates for custom post types:

Single Template

Create single-{post_type}.php for custom post types: 

<!-- single-product.php -->
<?php
get_header();
$price = get_custom_field('price', $post['id']);
$sku = get_custom_field('sku', $post['id']);
?>

<article class="product">
    <h1><?= esc($post['title']) ?></h1>
    
    <?php if ($image = get_featured_image($post['id'])): ?>
        <img src="<?= esc($image['url']) ?>" alt="<?= esc($post['title']) ?>">
    <?php endif; ?>
    
    <div class="product-meta">
        <?php if ($price): ?>
            <p class="price">$<?= esc($price) ?></p>
        <?php endif; ?>
        <?php if ($sku): ?>
            <p class="sku">SKU: <?= esc($sku) ?></p>
        <?php endif; ?>
    </div>
    
    <div class="content">
        <?= $post['content'] ?>
    </div>
</article>

<?php get_footer();

Archive Template

<!-- archive-product.php -->
<?php
get_header();
$products = Post::query([
    'post_type' => 'product',
    'status' => 'published',
    'limit' => 12
]);
?>

<div class="products-grid">
    <?php foreach ($products as $product): ?>
        <div class="product-card">
            <h3><?= esc($product['title']) ?></h3>
            <a href="<?= getPostUrl($product) ?>">View Product</a>
        </div>
    <?php endforeach; ?>
</div>

<?php get_footer();

Best Practices

Security

  • Always escape output — Use esc() for all dynamic content
  • Sanitize input — Never trust user data
  • Use CSRF tokens — Protect forms with csrfToken()

Performance

  • Minimize queries — Cache expensive database calls
  • Optimize images — Use appropriate thumbnail sizes
  • Minify assets — Compress CSS and JavaScript

Code Organization

  • Use functions.php — Keep logic separate from templates
  • Comment your code — Document complex sections
  • Follow naming conventions — Use consistent file and variable names

Accessibility

  • Semantic HTML — Use proper heading hierarchy
  • Alt text — Always provide image descriptions
  • Keyboard navigation — Ensure all interactive elements are accessible
